# 👥 CRM Модуль — Управление клиентами

## Обзор

CRM модуль включает:
- База клиентов с контактами
- Управление локациями (адреса, геозоны)
- Лиды и воронка продаж
- История взаимодействий

---

## 📋 Клиенты

### Страница клиентов
**URL:** `/crm/clients`

## 📍 Локации и геозоны

### Автогеокодирование
При добавлении адреса система автоматически:
1. Геокодирует адрес через Nominatim API
2. Сохраняет координаты (lat, lng)
3. Создаёт геозону для проверки

### Геозона
- **Радиус:** 500 метров от точки
- **Использование:** Проверка геолокации работников
- **Настройка:** Можно уточнить координаты вручную

## 👤 Контакты

### Мульти-контакты
Каждый клиент может иметь до 5 контактных лиц:

```typescript
interface Contact {
  name: string;
  role?: string;        // Должность
  phone?: string;
  email?: string;
  telegram?: string;
  isPrimary: boolean;   // Основной контакт
}
```

---

## 💰 Баланс клиента

### Журнал операций (Client Ledger)
Автоматически учитывает:
- **Дебет:** Рабочие сессии (начисления)
- **Дебет:** Закупки (расходы)
- **Кредит:** Оплаты от клиента

### Триггеры
- `onWorkSessionUpdate` → Добавляет запись при завершении сессии
- `onReceiptUpdate` → Добавляет запись при подтверждении чека

## 💾 Структура данных

### Client
```typescript
{
  id: string;
  name: string;
  type?: 'company' | 'individual';
  
  // Контакты
  contacts: Contact[];       // До 5 контактов
  
  // Локации
  locations: Location[];
  primaryLocationId?: string;
  
  // Финансы
  balance: number;
  totalSpent: number;
  totalHours: number;
  
  // Метаданные
  services?: string[];       // Связанные услуги
  notes?: string;
  
  status: 'active' | 'inactive' | 'archived';
  createdAt: Timestamp;
  updatedAt: Timestamp;
}
```

### Location
```typescript
{
  id: string;
  name: string;          // "Офис", "Склад"
  address: string;
  
  // Геокодирование
  coordinates?: {
    lat: number;
    lng: number;
  };
  geocodedAt?: Timestamp;
  
  isPrimary: boolean;
}
```
