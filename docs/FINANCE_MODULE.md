# 💰 Модуль финансов

## Обзор

Модуль финансов обеспечивает:
- Расчёт зарплат на основе отработанного времени
- Управление почасовыми ставками
- Финансовые отчёты по клиентам и сотрудникам
- Учёт закупок и возмещений

---

## 📊 Расчёт зарплаты

### Формула
```
Заработок = Отработанные часы × Почасовая ставка
```

### Приоритет ставок
Система использует следующий приоритет при определении ставки:

```
1. task.hourlyRate     — ставка из задачи (если указана)
2. user.hourlyRate     — ставка пользователя  
3. employee.hourlyRate — ставка из справочника сотрудников
4. 0                   — по умолчанию
```

### Где настроить ставку
**Веб-панель:** `/admin/employees` → Редактировать сотрудника → Hourly Rate

## 📈 Отчёты

### Страница Time Tracking
**URL:** `/crm/timetracking`

Показывает:
- Все рабочие сессии
- Время работы по сотрудникам
- Заработок за период
- Фильтры по датам и сотрудникам

### Страница Finance
**URL:** `/crm/finance`

Показывает:
- Сводка по зарплатам
- Корректировки и вычеты
- Расчётные периоды

---

## 🧾 Вкладка "Чеки"

**URL:** `/crm/shopping` → Вкладка "💰 Чеки"

### Три режима отчёта

1. **По клиенту** — расходы сгруппированы по клиентам
2. **По сотруднику** — кто сколько потратил
3. **Аудит** — полный список чеков с деталями

### Фильтры
- По дате (от-до)
- По статусу (все/подтверждённые/ожидающие)

### Карточки сводки
```
┌─────────────────┬─────────────────┬─────────────────┐
│  💵 Всего       │  ✅ Подтверждено │  ⏳ Ожидает     │
│  $2,450.00      │  $1,890.00       │  $560.00        │
└─────────────────┴─────────────────┴─────────────────┘
```

## 💾 Структура данных

### WorkSession (Рабочая сессия)
```typescript
{
  id: string;
  userId: string;
  clientId: string;
  startTime: Timestamp;
  endTime?: Timestamp;
  hourlyRate: number;      // Ставка на момент сессии
  totalEarnings: number;   // Рассчитанный заработок
  status: 'active' | 'completed' | 'cancelled';
}
```

### Receipt (Чек)
```typescript
{
  id: string;
  userId: string;
  clientId: string;
  amount: number;
  status: 'pending' | 'confirmed' | 'rejected';
  receiptPhotoUrl: string;
  goodsPhotoUrl?: string;   // Double Proof
  paymentSource?: string;   // Источник оплаты
  costCenter?: string;      // Центр затрат
  createdAt: Timestamp;
}
```

## 🔧 Технические детали

### Триггеры Firebase
- `onWorkSessionCreate` — начисление при создании сессии
- `onWorkSessionUpdate` — пересчёт при изменении
- `onReceiptUpdate` — обновление баланса клиента

### Firestore коллекции
- `workSessions` — рабочие сессии
- `employees` — справочник сотрудников (с hourlyRate)
- `receipts` — чеки закупок
- `clientLedger` — журнал операций по клиентам
