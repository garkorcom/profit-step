# 🚀 docs/ — Улучшения Документации

## 🔴 Критические

### 1. Синхронизация с кодом
Документация может быть устаревшей. Проверить каждый файл:

| Файл | Статус | Проблема |
|------|--------|----------|
| `CRM_MODULE.md` | ⚠️ Проверить | Возможно не отражает ERP v4 |
| `FINANCE_MODULE.md` | ⚠️ Проверить | Может не включать Bank Statements AI |
| `GTD_TASK_CREATION.md` | ⚠️ Проверить | GTD Create Page сильно изменился |
| `TELEGRAM_BOT.md` | ⚠️ Проверить | OpenClaw миграция |
| `USER_GUIDE.md` | ⚠️ Проверить | Новые страницы? |
| `SHOPPING_MODULE.md` | ✅ Вероятно ОК | Модуль стабилен |
| `TECHNICAL_API.md` | ⚠️ Проверить | Agent API v4? |
| `TIME_TRACKING.md` | ✅ Вероятно ОК | Модуль стабилен |

### 2. Убрать legacy папку
`docs/legacy-nov2025/` — данные на ноябрь 2025. Перенести в `_archived/docs/`.

## 🟡 Среднесрочные

### 3. API Reference (авто-генерация)
Использовать TypeDoc для автоматической генерации документации из кода:
```bash
npx typedoc --out docs/api src/api
```

### 4. Changelog
Вести `CHANGELOG.md` с крупными изменениями по датам.

### 5. Единый формат
Все документы должны следовать шаблону:
```markdown
# Название модуля

## Описание
Что делает этот модуль

## Ключевые файлы
| Файл | Назначение |

## Firestore коллекции
| Коллекция | Поля |

## API endpoints / Callable functions

## Связи с другими модулями

## Частые проблемы и решения
```

## 🟢 Долгосрочные

### 6. Wiki
Перенести документацию из markdown файлов на GitHub Wiki или Notion для удобной навигации.

### 7. Architecture Decision Records (ADR)
Фиксировать архитектурные решения:
- Почему CRA, а не Vite
- Почему Gemini + Claude (не один провайдер)
- Почему Firestore, а не Postgres
