# Категория: Tasks & Work (Задачи и Работа)

## Назначение
Модуль для управления рабочим процессом (GTD), календарем, учета рабочего времени сотрудников и управления закупками (Shopping). Это ядро всей продуктивности компании. Включает в себя 'Cockpit View' и Smart Task Constructor v3.

## Как работать (Сценарии пользователя)
- **GTD Board (Kanban)**: Создание новых задач в Inbox, перенос их в `Next Action`, `Waiting`, `Completed`. Назначаются ответственные и прикрепляются клиенты/проекты.
- **Calendar**: Просмотр встреч и дедлайнов. Оптимизирован для планшетов (iPad Pro).
- **Time Tracking**: Сотрудник нажимает "Start", стартует таймер. Таймер связывается с задачей. Можно поставить на паузу или завершить смену. Оплата считается исходя из `hourlyRate`.
- **Shopping**: Составление списков покупок, которые потом могут быть закрыты через бота или превращены в `Purchase Orders` / Расходы (`Costs`).

## Основные Связи (Отношения с другими модулями)
- **Time Tracking ↔ Finance**: Завершенные сессии таймтрекера (Work Sessions) автоматически генеруют начисления в `Payroll` (Зарплаты) в блоке Finance.
- **Tasks ↔ Estimator**: Созданная смета (`Estimate`) может быть автоматически разбита на GTD задачи при передаче в работу.
- **Tasks ↔ CRM**: Вы можете прикрепить к задаче конкретного клиента.
- **Tasks ↔ OpenClaw (Агенты)**: Бот (Smart Dispatcher) может слушать голосовое сообщение в Telegram и автоматически создавать задачу в Inbox. Возможна синхронизация через API (`functions/src/agent/agentApi.ts`).

## Как это устроено (Для разработчика)
- **Коллекции Firestore**: 
  - `gtd_tasks`: Основная таблица задач.
  - `work_sessions`: Сессии учета времени. Статусы: `active`, `paused`, `completed`.
- **Файлы/Компоненты**:
  - `src/components/tasks-unified/` (Smart Task Constructor).
  - `src/components/time-tracking/` (TimeTracker/Cockpit View).
- **Архитектура**: 
  - Разработана в концепции "One Page Everywhere" на базе контекста React. 
  - Пагинация и фильтрация идут как query params (`?view=board&status=inbox`).
