# Категория: Dashboard (Дашборд)

## Назначение
Главная панель управления (Dashboard) служит точкой входа и агрегатором ключевых метрик всей CRM. Она обеспечивает моментальный обзор состояния бизнеса (Profit & Loss, количество сделок, статистика задач, активные смены).

## Как работать (Сценарий пользователя)
1. Пользователь заходит на `/admin/dashboard`.
2. В зависимости от уровня доступа (RBAC), он видит сводные графики: либо только свои задачи/смены (Role: Worker), либо глобальную аналитику по всей компании (Role: Admin).
3. Интерактивные чарты (обычно Recharts или Chart.js) позволяют кликнуть и провалиться в конкретный модуль (например, Finance).

## Основные Связи (Отношения с другими модулями)
Дашборд не хранит собственные данные, он **агрегирует** их:
- С **Finance**: для вычисления Profit & Loss (доходы и расходы), вывода текущих затрат по категориям.
- С **Tasks & Work**: для вывода количества открытых/завершенных задач (Inbox, Next Action).
- С **CRM (Deals/Clients)**: для показа воронки продаж и новых клиентов.
- С **Time Tracking**: для показа активных сессий (Cockpit View).

## Как это устроено (Для разработчика)
- **Файлы/Компоненты**: `src/pages/admin/AdminDashboardPage.tsx`
- **Состояние**: Данные подтягиваются через хуки `useCollection` или `useQuery` напрямую из Firestore (`users`, `costs`, `gtd_tasks`, `deals`).
- **Сложности**: Возможны проблемы с производительностью при росте базы (N+1 запросы). Для сложных метрик (визуализация P&L за год) рекомендуется использовать агрегацию через *Data Warehouse (BigQuery)*, а не тянуть тысячи документов на клиент.
