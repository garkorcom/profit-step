Авторизация — только 2 режима в проде

Проверено против functions/src/agent/agentMiddleware.ts на 2026-04-16. Остальные описания в старых доках (docs/AGENT_SETUP_GUIDE.md, crm_api/*) либо помечены как LEGACY, либо описывают незапущенные планы.

РежимДля когоHeader
Static Master Token
AGENT_API_KEY (env)		 Серверные интеграции, openclaw-боты, скрипты Authorization: Bearer <token>
Firebase ID Token (JWT) Браузерный фронтенд (https://profit-step.web.app) Authorization: Bearer <firebase_id_token>

Любой третий вариант (например «per-employee 40-hex token» из старой документации) не работает — сервер вернёт 401.

Опциональные заголовки

  • X-Source: agent — для пометки в audit log
  • X-Idempotency-Key: <uuid> — защита от дублей при POST
  • X-Impersonate-User: <firebase_uid> — master-токен действует от имени сотрудника (применяются его RLS + role scopes)

Настройка openclaw.ai

  1. В Bot settings → Secrets добавь переменную PROFIT_STEP_TOKEN со значением мастер-токена, который выдаёт Денис.
  2. В HTTP-tool бота в секции Headers пропиши: 
    Authorization: Bearer ${secrets.PROFIT_STEP_TOKEN}
Content-Type: application/json
  3. В System Prompt бота скопируй целиком блок SYSTEM PROMPT со вкладки «System Prompt» на этой странице.

Smoke-test

Перед подключением бота проверь, что токен живой:

export PROFIT_STEP_TOKEN="<мастер-токен-от-дениса>"

# 1. /health без авторизации — должен ответить 200
curl -s https://profit-step.web.app/api/health | jq

# 2. С плохим токеном — должен ответить 401
curl -s -H "Authorization: Bearer fake" \
     https://profit-step.web.app/api/dashboard | jq
# Ожидаемо: {"error":"Invalid authorization token"}

# 3. С правильным токеном — должен отдать JSON дашборда
curl -s -H "Authorization: Bearer $PROFIT_STEP_TOKEN" \
     https://profit-step.web.app/api/dashboard | jq '. | keys'
# Ожидаемо: список ключей (activeSessions, tasksDueToday, ...)
Если шаг (3) вернул 401 — токен устарел или не тот. Попроси Дениса выдать актуальный AGENT_API_KEY (он же PROFIT_STEP_TOKEN для openclaw). Не пытайся угадать токен и не проси пользователя его ввести — это секрет сервера.
